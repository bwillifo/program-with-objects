Lesson 1: The Basics of the Web and HTML

History and web basics

Invented in the early 1990's
Basics include: Web, HTML, URL, HTTP, Web Applications
The World Wide Web is a collection of HTML (Hypertext Markup Language) Documents by which HTML "glues" Documents, Images, videos together and Hyperlinks or "links" makes up the web

Major Pieces

- You, your computer, internet, HTTP and Servers make up the major pieces
- Browsers such as Internet Explorer, Chrome, Firefox use HTTP to communicate with the servers.

Key Definitions

- HTTP = Main protocol of the web.
- Servers = hosts the files that make up the web
- Internet = Largest computer network
- Browser = program on your computer to display the files on the web.
- HTML = (Hypertext Markup Languange) is the heart of the web; takes text content and by markup to what it looks like on the web

Details

HTML Markup- An element is everything within a set of opening and closing tags. Tags help define key attributes of the element. Tags can determine the text layout. Including Text bold tag b; italic tag em. Void Tags do not have a close tag. Tag br is a line break HTML Attributes (ATTR) Always a value Anchor tag a includes: attribute; value; content; closing tag. tag img is images to be embedded. tag p paragraph Inline vs. Block Block has a height and width and can be managed with variations/distinctions around the text. Inludes tag p Inline impacts the normal line items. Word within a sentence."nothing fancy".
- Inline Includes tags b, em, img, a, span, strong
- Block includes tags div, p, for span vs. div span works with inline portion of a document. div works with few paragraphs or images with captions

Lesson 3: Adding CSS Style to HTML Structure

CSS Definition and concept

Cascading Style Sheets (CSS) with structure based on boxes, CSS helps focus on style and appearance. Seperation of HTML and CSS help seperate the formatting from the HTML coding. Cascading allows for replication within the coding to make a change in one location but touch multiple areas. This allows coders to avoid mistakes by typing code multiple times. In a manner of Cascading styles, CSS goes down and down into the code cascading (like National->state->municipal->home laws) until the best rule for an element gets applied. Cascade also can be referenced as inheritance.

Repetition

Avoiding repetition is important. Failing to avoid repetition leads to adding code in various places with little differences. This may lead to errors which takes considerable amount of time to research. If a coder decides to make changes having Consistency is key. Consistency is the concept to keep details looking the same. To keep code the same, an example is to assign a tag div the same class and then specify the style for that class allowing coders to make a single line change that impacts an entire class association throughout the code. One line changes multiple areas.

Selectors

Selectors in CSS defines which particular element ion a page a particular style will be applied to. To define specific elements to apply style select elements by Class. This allows CSS to (example: .desription { attribute: color; }. Within the HTML the tag div that has a Class of desciption can be turned red (attribute : red;). CSS has a lot of functionality. Use %based or hex code for colors.

Box Model

Box Model consist of Margins, border, padding and content. Contents is the nucleus surrounding by the other components. WOrking wiht Box Model helps structure the boxes for good appearance and makes a "Clean" Website. Key use box-sixing: border-box; helps standardize and no thave to recalculate boxes each time. Browser may be different to use webkit, mox, ms for box sizing. Class { Display: Flex; } will help change display of parent container."
